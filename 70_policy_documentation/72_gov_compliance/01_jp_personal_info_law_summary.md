# jp_personal_info_law_summary.md

## Objective / 目的

- Understand the key principles and obligations under Japan’s Act on the Protection of Personal Information (APPI).
- Identify how these requirements affect corporate information security operations and internal policies.
- Prepare a foundation for mapping APPI requirements to ISMS controls and other compliance standards.
- 日本の個人情報保護法（APPI）の基本原則および義務を理解する。
- これらの要件が企業の情報セキュリティ運用や社内規程にどのように影響するかを把握する。
- APPIの要件をISMS管理策や他のコンプライアンス基準と対応付ける基礎を整える。

## Scenario / シナリオ

- The compliance team requests a concise summary of Japan’s personal data protection requirements in preparation for an ISMS re-certification audit.
- IT team must highlight the legal obligations, key definitions, and enforcement mechanisms relevant to internal operations.
- コンプライアンスチームから、ISMS再認証監査に向けた日本の個人情報保護法の簡潔な要点まとめを依頼されました。
- 法的義務、重要な定義、および社内運用に関わる執行メカニズムを整理します。

## Impact on Internal Data Handling / 社内データ取扱いへの影響

| APPI Requirement / APPI 要件                | Internal Impact / 社内影響                                                                                                                                           |
| ----------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Personal Information Definition / 個人情報の定義 | Identify which internal records contain personal data. / 社内でどの記録が個人情報に該当するかを特定する。                                                                                |
| Purpose Limitation / 利用目的の制限              | Ensure data is collected and used only for documented business purposes. / データは文書化された業務目的の範囲内で収集・利用する。                                                           |
| Accuracy / 正確性の確保                         | Implement procedures for updating and correcting personal data. / 個人情報の更新・訂正手順を整備する。                                                                             |
| Data Security / 安全管理措置                    | Apply technical (encryption, access control) and organizational measures to prevent leakage or unauthorized access. / 漏洩・不正アクセス防止のため、技術的（暗号化、アクセス制御）および組織的対策を実施。 |
| Retention Limitation / 保管期間の制限            | Define retention schedules and secure deletion procedures. / 保管期間ルールを定め、安全な廃棄手順を実施する。                                                                            |
| Consent & Notification / 同意・通知            | Establish workflows for obtaining consent or notifying individuals for secondary use. / 二次利用に必要な同意取得や通知フローを構築する。                                                 |
| Data Subject Rights / 個人の権利対応             | Implement processes to handle disclosure, correction, deletion, or suspension requests. / 開示・訂正・削除・利用停止請求への対応手順を整備する。                                            |
| Cross-Border Transfer / 国際データ移転           | Verify protection measures when transferring data to foreign services or offices. / 海外サービスや拠点へのデータ移転時の保護措置を確認する。                                                 |
