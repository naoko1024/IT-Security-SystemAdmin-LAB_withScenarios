# 01_internal_audit_report_sample.md

## Objective / 目的

- This document provides a sample internal audit report structure based on ISMS requirements, illustrating how audit results can be documented and communicated within an organization.  
	本書は、ISMS要件に基づく内部監査報告書のサンプル構成を示し、監査結果をどのように記録・共有できるかを示します。

---

## Audit Scope / 監査範囲

- Period: FY2024 Q4
- Departments: IT Department, General Affairs
- Focus Areas: Access Control, Data Protection Policy Compliance
    
- 対象期間：2024年度第4四半期
- 対象部門：IT部門、総務部門
- 対象領域：アクセス制御、データ保護方針の遵守状況
---

## Audit Summary / 監査概要

|Item / 項目|Description / 内容|
|---|---|
|Audit Type / 監査種別|Regular Internal Audit / 定期内部監査|
|Audit Team / 監査チーム|2 members from ISMS Secretariat / ISMS事務局より2名|
|Method / 方法|Document review, interviews, and on-site inspection / 文書レビュー、ヒアリング、現地確認|

---

## Findings Summary / 指摘事項概要

| Category / 区分                | Finding / 指摘内容                                                             | Root Cause / 原因                                                              | Impact / 影響                                                                                                          |     |
| ---------------------------- | -------------------------------------------------------------------------- | ---------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- | --- |
| Documentation / 文書管理         | Access control procedure not updated after system change / システム変更後の手順書が未更新 | Change management process was not triggered; no version control established  | Medium / 中<br><br>Potential for incorrect access assignments or unreviewed access privileges / 誤ったアクセス付与や権限見直し漏れの可能性 |     |
| Incident Response / インシデント対応 | No post-incident review report prepared / 事後報告書の作成漏れ                       | Incident handling workflow lacked closure step; responsible role not defined | High / 高<br><br>Lost learning opportunity; repeat incidents possible / 教訓化されず再発の可能性                                  |     |

---

## Recommendations / 推奨事項

- Update the access control procedure to reflect the current environment.  
    　現行システム環境に合わせてアクセス制御手順書を更新すること。
- Establish a review process after each incident.  
    　インシデント発生後にレビューを行うプロセスを確立すること。

---

## Audit Conclusion / 監査結論

- The audit confirmed general compliance with ISMS controls, with a few improvement points identified for process documentation and incident review.  
	全体としてISMS管理策への準拠は確認されたが、文書化および事後レビューに関して改善の余地があると判断された。
